# Encapsulamiento

getter y setter
Le pondremos un candado a la informacion y creamos "Ventalinllas de atencion" (Getters y setters) con guardias de seguridad que validaran quien entra y que datos se puede modificar

**Sintaxis** Para converir un atributo publico en python en **Privado** (Invisible para el mundo exterior), Simplemente le colocamos dos guiones bajos ('__') al inicio de du nombre dentro del constructor.

```python

class Cuenta:
    def __int__(self, saldo_inicial):
        self.nombre = 'Publica'
        self.__saldo = saldo_inicial
        
mi_cuenta = Cuenta(1000)

print(mi_cuenta.nombre)
print(mi_cuenta.__saldo)
```

## las ventanillas de atencion: getter y Setter (metodos tradicionales)
Constructor metodos publicos especificos llamados *Getters* (Obtenedores) y *Setters* (configuradores).
* **Getters** Es un metodo que solo sirve para retornar ('Return') 

```python
class UsuarioBancario:
    def __init__(self, nombre_ingresado, pin_ingresado):
        self.nombre = nombre_ingresado
        #atributo privado
        self.__pin = pin_ingresado

    #get tradicionales (ventanilla para consultor)
    def get_pin(self):
        return self.__pin 

    def set_pin(self, nuevo_pin):
        #validacion
        if len(str(nuevo_pin)) == 4:
            self.__pin = nuevo_pin
            print('Operacion exitosa Pin actualizados.' )
        else: 
            print('Error: el PIN debe tener exactamente 4 digitos')

#Programa principal
print('--- Sistema de seguridad ---')
cliente = UsuarioBancario('Diego', 1234)

#Primer intento de Hackeo: intento directamente al atributo privado
print(cliente.__pin)
```
